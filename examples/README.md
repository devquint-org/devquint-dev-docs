# Примеры планов по доменам

Выбери похожий на твой проект и скопируй структуру как стартовую точку.

---

## Доступные примеры

| Домен | Файл | Для кого |
|-------|------|---------|
| **Backend API** | [`backend.md`](backend.md) | REST/GraphQL, микросервисы, Node/Python/Java |
| **Frontend** | [`frontend.md`](frontend.md) | React/Vue/Angular, веб-приложения |
| **Microservices** | [`infra.md`](infra.md) | Docker, Kubernetes, распределённые системы |
| **Mobile** | [`mobile.md`](mobile.md) | React Native, Flutter, iOS, Android |

---

## Как использовать пример

1. **Прочитай пример** похожий на твой проект
2. **Скопируй структуру этапов** из примера
3. **Адаптируй под свой проект:**
   - Переименуй этапы если нужно
   - Добавь/удали подэтапы
   - Измени условия завершения
4. **Используй шаблон** → [`templates/plan-template.md`](../templates/plan-template.md)
5. **Проверь чеклист** → [`core/CHECKLIST.md`](../core/CHECKLIST.md)

---

## Примеры охватывают

- ✅ Структура этапов (какие фазы)
- ✅ Таблица зависимостей (от чего что зависит)
- ✅ Условия завершения (когда этап готов)
- ✅ Параллельная разработка (кто может работать вместе)
- ✅ Архитектурные решения (почему так)
- ✅ Риски (на что обратить внимание)

---

## Если твой домен не в списке

Используй **Backend** как базис для большинства случаев - логика одна и та же:

```
1. Infrastructure
2. Models/Schema
3. Core logic
4. Integration layer
5. API/Interface
6. Advanced features
7. Testing
8. Deployment
```
